# GameHub - Cloudflare R2 Storage Design (Phase 4)

## Overview

Cloudflare R2 is used for all file storage in GameHub. It provides S3-compatible API with excellent latency and no egress fees. This document describes the storage structure, key naming conventions, and operational patterns.

## Storage Architecture

### Key Principles

1. **Presigned URLs Only**: Never expose R2 credentials to the frontend
2. **Direct Upload**: Admin client uploads directly to R2 using presigned PUT URLs
3. **Direct Download**: Public files accessible via public CDN URLs
4. **Prefix-based Operations**: Use prefixes for bulk operations (delete, list)
5. **Public Base URL**: Configure R2 public domain for CDN delivery

### Bucket Configuration

```
Bucket Name: gamehub
Region: Auto (Global)
Public Access: Enabled (for public CDN delivery)
CORS: Configured for admin-web and launcher origins
```

**Environment Variables Required:**

```env
# API Configuration
R2_ACCOUNT_ID=a4652a07eb284b5295b7e710ff59a071
R2_BUCKET=gamehub
R2_ACCESS_KEY_ID=<from R2 API token>
R2_SECRET_ACCESS_KEY=<from R2 API token>
R2_PUBLIC_BASE_URL=https://cdn.example.com  # Custom domain recommended

# For launcher updates
LAUNCHER_UPDATE_BASE_URL=https://cdn.example.com/launcher-updates
```

## Key Structure

### 1. Game Images

**Cover Image** (landscape, ~16:9 aspect):
```
games/{gameId}/cover/cover.webp
```

**Banner Image** (landscape, wide aspect):
```
games/{gameId}/banner/banner.webp
```

**Example:**
```
games/507f1f77bcf86cd799439011/cover/cover.webp
games/507f1f77bcf86cd799439011/banner/banner.webp
```

**Operations:**
- Upload: Admin presigns PUT URL, client uploads directly
- Access: Public via `R2_PUBLIC_BASE_URL/games/{gameId}/cover/cover.webp`
- Delete: When game is deleted, entire `games/{gameId}/` prefix is deleted

---

### 2. Patch Files

**Individual Patch File**:
```
games/{gameId}/versions/{patchVersionId}/files/{relativePath}
```

**Directory Structure** (reflects game filesystem):
```
games/507f1f77bcf86cd799439011/versions/507f191e810c19729de860ea/files/
├── data/
│   ├── localization/
│   │   ├── vietnamese.pak
│   │   └── english.pak
│   └── models/
│       └── character.fbx
├── config.ini
└── patch.txt
```

**Example Keys:**
```
games/507f1f77bcf86cd799439011/versions/507f191e810c19729de860ea/files/data/localization/vietnamese.pak
games/507f1f77bcf86cd799439011/versions/507f191e810c19729de860ea/files/config.ini
```

**Operations:**
- Upload: Multi-part presigned URLs for each file
- Access: Public URLs in manifest for launcher download
- Delete: Delete entire version prefix after replacement or archival
- Verify: List prefix to verify all files uploaded

**Manifest File** (JSON metadata for patch):
```
games/{gameId}/versions/{patchVersionId}/manifest.json
```

**Example:**
```
games/507f1f77bcf86cd799439011/versions/507f191e810c19729de860ea/manifest.json
```

## Operational Workflows

### 1. Upload Game

```
Admin uploads cover/banner images
  ↓
API presigns PUT URLs
  ↓
Client uploads to R2 directly
  ↓
API stores media URLs in MongoDB Game.coverImage, Game.bannerImage
  ↓
Public URLs: https://cdn.example.com/games/{gameId}/cover/cover.webp
```

### 2. Create & Upload Patch

```
Admin selects game + folder with patch files
  ↓
API creates PatchVersion (DRAFT)
  ↓
Client computes SHA-256 for each file
  ↓
API presigns PUT URLs for all files
  ↓
Client uploads all files to R2 in parallel
  ↓
Client calls completeUpload with file metadata
  ↓
API creates PatchFile records in MongoDB
  ↓
API/Worker creates & uploads manifest.json to R2
```

### 3. Publish Patch

```
Admin clicks Publish
  ↓
API builds PatchManifest JSON
  ↓
API uploads manifest to R2: games/{gameId}/versions/{patchVersionId}/manifest.json
  ↓
API sets PatchVersion.status = PUBLISHED
  ↓
API updates Game.latestPatchVersionId = new version
  ↓
Launcher detects new patch, downloads manifest, then downloads files
```

### 4. Replace Patch (with Delete Old R2 Files)

```
Admin creates new PatchVersion with mode=REPLACE_EXISTING
  ↓
Admin uploads files (same as Create & Upload)
  ↓
Admin publishes new patch
  ↓
API marks old patch.status = REPLACED
  ↓
Worker job triggered: delete-patch-r2-prefix
  ↓
Worker deletes games/{gameId}/versions/{oldPatchVersionId}/ prefix
  ↓
Frees up R2 storage
```

### 5. Delete Game

```
Admin clicks Delete
  ↓
API sets Game.status = DELETING, Game.deletedAt = now
  ↓
Worker job triggered: delete-game-r2-prefix
  ↓
Worker lists all objects under games/{gameId}/
  ↓
Worker deletes in batches of 1000
  ↓
Worker sets Game.status = DELETED
  ↓
Game is now completely removed from R2 + MongoDB
```

### 6. Launcher Auto-Update

```
Admin uploads new launcher artifacts (exe, dmg, AppImage)
  ↓
CI/CD generates latest.yml via electron-builder
  ↓
Admin presigns artifacts via API
  ↓
CI/CD uploads exe/dmg/AppImage + latest.yml to launcher-updates/{platform}/
  ↓
Admin publishes release in API
  ↓
API sets LATEST_LAUNCHER_VERSION env (via Railway dashboard or config)
  ↓
Launcher startup checks /launcher/config
  ↓
If currentVersion < minSupportedVersion → Force Update Modal
  ↓
electron-updater downloads latest.yml → verifies hash → downloads exe
  ↓
Launcher calls quitAndInstall
```

## Presigned URL Flow

### For Upload (Admin Client → R2)

```
1. Admin selects files
2. Client calls API: POST /admin/patches/{id}/presign-files
   Body: { files: [{ relativePath, size, sha256, contentType }] }
3. API calls R2Service.presignPut() for each file
   Returns: { uploadUrl, r2Key, expiresAt }
4. Client receives presigned URLs (expires in 1 hour)
5. Client uploads files directly to presigned URLs (no API involved)
6. Client calls API: POST /admin/patches/{id}/complete-upload
   Body: { files: [{ relativePath, r2Key, size, sha256, contentType }] }
7. API stores metadata in MongoDB PatchFile collection
```

**Important**: Presigned URLs expire after 1 hour. If upload takes longer, request new URLs.

### For Download (Public)

```
1. Launcher downloads manifest.json from R2 (public URL)
2. Manifest contains public URLs for each patch file
3. Launcher downloads files from public URLs (no authentication)
4. Launcher verifies SHA-256 hash
5. Launcher installs files locally
```

## Cost Optimization

1. **Delete Unused Patches**: Archive or delete old patch versions after new patches published
2. **Cleanup Expired Uploads**: UploadSession TTL auto-deletes incomplete uploads
3. **Use Public URLs**: CDN caching for popular patch files
4. **Compress Files**: Reduce patch file sizes with compression
5. **Monitor Quota**: Set CloudFlare alerts for storage usage

---

## Security Best Practices

1. **Never Expose R2 Credentials**: All operations go through API presigned URLs
2. **Presigned URL Validation**: Verify expiresAt before trying to use
3. **CORS Configuration**: Restrict origins for cross-origin requests
4. **Bucket Versioning**: Optional, enables rollback for critical files
5. **Access Logs**: Enable R2 access logs for audit trail
6. **IP Allowlisting**: Restrict API server IP for R2 credentials

## API Endpoints Using R2

### Game Media

```
POST   /admin/games/:id/cover/presign        → Presign cover image upload
POST   /admin/games/:id/banner/presign       → Presign banner image upload
PATCH  /admin/games/:id/media                → Save media URLs after upload
```

### Patch Upload

```
POST   /admin/patches/:id/presign-files      → Presign patch files
POST   /admin/patches/:id/complete-upload    → Confirm upload + create manifest
POST   /admin/patches/:id/publish            → Generate + upload manifest.json
GET    /launcher/patches/:id/manifest        → Public: Get manifest
```

### Launcher Artifacts

```
POST   /admin/launcher/releases/:id/presign-artifacts  → Presign artifact upload
POST   /admin/launcher/releases/:id/publish            → Mark as published
GET    /launcher/config                                 → Get download URLs
```

## Troubleshooting

### "Cannot find module" after build

**Solution**: Clear dist, rebuild:
```bash
pnpm --filter @gamehub/api exec rimraf dist
pnpm --filter @gamehub/api build
```

### Presigned URL returns 403

**Causes**:
- URL expired (check expiresAt)
- Content-Type mismatch
- R2 credentials invalid
- Bucket doesn't exist

**Solution**: Request new presigned URL, verify credentials

### Manifest download 404

**Solution**:
1. Verify manifest key is correct: `games/{gameId}/versions/{patchVersionId}/manifest.json`
2. Check patch status is PUBLISHED
3. Verify R2 public access enabled
